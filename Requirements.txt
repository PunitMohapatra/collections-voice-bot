## PROJECT OVERVIEW
A voice bot for a loan collections company that:

1. Calls customers and reminds them of overdue loan amounts (fetched live from database)
2. Communicates in English and Hindi language
3. Captures Promise to Pay (PTP) — date and amount — with business rule validation
4. Answers customer questions about their loan in real time
5. Logs every call outcome as a followup record
6. Escalates to human agent when customer disputes

## ARCHITECTURE — 4 LAYERS
Layer 1 — PostgreSQL: stores all data
Layer 2 — Spring Boot API: exposes 4 REST endpoints to the bot
Layer 3 — Python/Rasa/FastAPI: bot brain and orchestration
Layer 4 — Whisper STT + Piper TTS: voice input/output

CRITICAL RULE: The bot NEVER reads from the database directly.
It always calls the Spring Boot API. This means company integration
later is just changing one URL.

## SPRING BOOT — 4 REST ENDPOINTS

### Endpoint 1: GET /api/v1/accounts/{accountNumber}
Returns full account details as AccountDTO.
AccountDTO fields: accountId, accountNumber, customerName, loanAmount,
overdueAmount, preferredLanguage, lastEmiDate, charges,
lastPaymentAmount, lastPaymentDate

### Endpoint 2: GET /api/v1/promise-policy
Returns the first row of promise_policy table.
Use: policyRepo.findTopBy().orElseThrow()
(NOT findFirst() — Spring Data does not support that method name)

### Endpoint 3: POST /api/v1/followup
Request body: { accountId, remarks }
Saves a new followup record and returns it with generated followupId.

### Endpoint 4: POST /api/v1/promise
Request body: { accountId, followupId, promiseAmount, promiseDate }
MUST validate before saving:

1. promiseDate must not exceed LocalDate.now().plusDays(maxPromiseDays)
2. promiseAmount must be >= overdueAmount * (minPromisePercent / 100)
Return 400 with message if validation fails.
Return 200 with saved promise if valid.

### ActionFetchAccount (action_fetch_account):

- Reads account_number slot
- Calls GET /api/v1/accounts/{accountNumber}
- Sets slots: account_id, customer_name, overdue_amount, charges, last_emi_date
- Uses timeout=5 on all requests
- Handles ConnectionError gracefully

### ActionValidateAndSavePTP (action_validate_and_save_ptp):

- Reads slots: ptp_date, ptp_amount, overdue_amount, account_id
- Calls GET /api/v1/promise-policy to get validation rules
- Validates date against maxPromiseDays (try multiple date formats)
- Validates amount against minPromisePercent
- If invalid: dispatcher.utter_message with explanation, return SlotSet(field, None)
- If valid: calls POST /api/v1/promise, sets ptp_saved=True

### ActionSaveFollowup (action_save_followup):

- Auto-generates remarks based on last_intent and ptp_saved slot
- Calls POST /api/v1/followup
- Sets followup_id slot from response

## FASTAPI BOT SERVER (bot/main.py)

### Key design decisions:

1. FastAPI fetches account data from Spring Boot DIRECTLY — does not rely on Rasa to fetch it
2. All slots are set via Rasa tracker events API before the conversation starts:
POST /conversations/{sender_id}/tracker/events
with list of slot events
3. The opening greeting is built directly in FastAPI from account data — Rasa is NOT called for the greeting message
4. Rasa tracker events are used to record the greeting happened so dialogue state is correct
5. For handle_message: send transcript to Rasa, get response, synthesise to audio

### Endpoints:
GET /health — returns status of all loaded models
POST /call/start?account_number=X — starts a call, returns WAV audio greeting
POST /call/message?account_number=X — receives WAV audio, returns WAV audio response

### Greeting logic in call/start:

- Fetch account from Spring Boot
- Set all slots via tracker events API
- Build greeting text directly:
Hindi (lang=hi): "Namaste, kya main {name} ji se baat kar sakta hoon?
Aapke loan account par {overdue} rupaye baaki hain
aur {charges} rupaye ke charges hain.
Aap kab tak payment kar sakte hain?"
English: "Hello, am I speaking with {name}?
Your loan account has an overdue amount of {overdue} rupees
with additional charges of {charges} rupees.
When would you be able to make this payment?"
- Record greeting in Rasa tracker as events (user greet + action events)
- Synthesise greeting text to WAV and return

### TTS synthesis — use synthesize_wav method:
voice.synthesize_wav(text, wav_file_object, set_wav_format=True)
where wav_file_object is opened with wave.open(path, "wb")

### Piper voice models:
Hindi: hi_IN-rohan-medium.onnx
English: en_US-amy-low.onnx
(downloaded from huggingface rhasspy/piper-voices)

### Whisper model: use "medium" size, device="cpu"
